#### 当前状态评估：⚠️ 需要优化

**当前案例：** 三名嫌疑人（独立/互斥/共同作案）的概率计算

**问题：**

2. "共同作案"的概率设定（50%）缺乏法律依据
3. 没有展示条件概率在证据链分析中的实际应用

**优化建议：**

**新案例：证据链的条件概率分析**

```
场景：某盗窃案有以下证据：
- 指纹证据：在作案工具上发现被告指纹（P=80%是被告留下）
- 监控证据：监控拍到类似身形的人（P=70%是被告）
- 证人证言：证人辨认被告（P=60%准确）

问题：如果这些证据独立，同时指向被告的概率是多少？
如果证据不独立（如证人是看到监控后才辨认），概率如何变化？
```

**统计概念：**
- 独立证据的联合概率：P(A∩B∩C) = P(A) × P(B) × P(C)
- 条件概率：P(A|B) ≠ P(A)
- 贝叶斯定理的初步应用

**法律意义：**
- 证据链的强度评估
- 独立证据vs累积证据的区别
- 避免"证据堆砌"谬误

### Chapter 3: 量刑分布 - 概率分布

#### 当前状态评估：❌ 必须修复

**当前问题：**
1. **使用正态分布模拟刑期是错误的！**
   - 刑期数据是右偏的（有下限0，无明确上限）
   - 正态分布允许负值，刑期不可能为负
   - 正态分布是对称的，而刑期通常右偏

2. **统计概念混淆：**
   - 展示的是"假设刑期服从正态分布"，而非真实数据
   - 需要明确说明这是"模拟数据"

**修复方案：改用Gamma分布**

```typescript
// Gamma分布参数
const gammaPDF = (x: number, shape: number, scale: number) => {
  // 使用Gamma分布，适合非负、右偏数据
  if (x <= 0) return 0
  return (Math.pow(x, shape - 1) * Math.exp(-x / scale)) / 
         (Math.pow(scale, shape) * gamma(shape))
}

// 不同罪名的参数
const crimeData = {
  theft: { name: '盗窃罪', shape: 2, scale: 12, unit: '月' },      // 均值24月
  fraud: { name: '诈骗罪', shape: 3, scale: 12, unit: '月' },      // 均值36月
  assault: { name: '故意伤害罪', shape: 4, scale: 12, unit: '月' }, // 均值48月
  corruption: { name: '贪污受贿罪', shape: 5, scale: 12, unit: '月' } // 均值60月
}
```

**优化后的案例描述：**
```
场景：分析某法院过去5年盗窃案件的刑期分布。

统计说明：
- 刑期数据使用Gamma分布模拟（非负、右偏）
- 实际司法数据中，大多数案件刑期较短，少数重刑案件形成右尾
- 此模拟用于教学目的，展示如何用概率分布理解量刑模式

法律意义：
- 识别量刑的集中趋势和离散程度
- 发现异常量刑案例（过长或过短）
- 评估量刑规范化政策的效果
```

**可视化改进：**
- 添加直方图（离散）vs 密度曲线（连续）的对比
- 标注均值、中位数、常见区间
- 显示真实案例的分布形状（右偏）

### Chapter 4: 置信区间 - 频率派推断

#### 当前状态评估：❌❌ 严重问题，必须修复

**当前问题（概念混淆）：**

1. **严重错误：将"置信区间"等同于"合理范围"**
   ```
   当前代码中的错误概念：
   "判断该法院量刑是否在该地区合理范围内（18-30个月）"
   
   绿色阴影区域标注为"合理量刑范围"
   ```

2. **概念混淆：**
   - 置信区间是**估计的不确定性范围**，不是**总体的合理范围**
   - 95%置信区间意思是：重复抽样100次，约95次区间包含真实均值
   - 不是"有95%的概率真实值落在这个区间"（贝叶斯解释）
   - 更不是"这个区间包含95%的数据点"

3. **错误的法律应用：**
   - 将置信区间上下限作为"量刑是否合规"的判断标准是错误的
   - 这会导致对统计概念的误解和司法实践中的误用

**正确概念：**
```
置信区间（Confidence Interval）：
- 是对"估计值不确定性"的量化
- 反映的是抽样误差，不是总体的"正常范围"
- 不能用来判断个体案件是否"合理"

正确应用示例：
"我们有95%的置信度认为，A法院的平均刑期在22-26个月之间"
而不是：
"22-26个月是合理量刑范围，超出就不合理"
```

**修复方案：**

**新案例：法院量刑水平的估计**

```
场景：某市有50个基层法院，司法局想评估各法院的平均量刑水平。
方法：从每个法院随机抽取100个已判决案件，计算平均刑期及其95%置信区间。

正确理解：
- 置信区间反映了"样本均值估计总体均值的不确定性"
- 区间宽窄取决于样本量和数据变异性
- 区间重叠不代表量刑一致，不重叠也不代表不一致（需假设检验）

可视化：
- 显示50个法院的置信区间
- 展示约95%的区间包含真实均值（模拟验证）
- 明确标注：这是估计的不确定性，不是量刑标准范围
```

**代码修改要点：**
```typescript
// 移除"合理范围"的错误概念
// 改为：估计的不确定性

const correctInterpretation = `
95%置信区间的正确理解：
- 含义：如果我们重复抽样很多次，大约95%的区间会包含真实的总体均值
- 不是：真实均值有95%的概率落在这个特定区间内（频率派观点）
- 不是：这个区间包含了95%的案件刑期
- 不是：超出这个区间的量刑就是"不合理"的
`;

// 可视化改进
// - 绘制多个置信区间
// - 标注包含真实均值的比例
// - 移除"合理范围"的阴影区域
```

**法律意义（修正后）：**
- 理解统计估计的不确定性
- 避免将置信区间误用为"标准范围"
- 正确评估法院间量刑差异的统计显著性

### Chapter 3: Gamma分布替换

```typescript
// 替换原来的正态分布函数
const gammaPDF = (x: number, shape: number, scale: number): number => {
  if (x <= 0) return 0;
  
  // 使用对数Gamma函数避免溢出
  const logGamma = (n: number): number => {
    if (n <= 0) return Infinity;
    // Lanczos近似或 Stirling近似
    if (n < 10) {
      let result = 0;
      for (let i = 2; i < n; i++) result += Math.log(i);
      return result;
    }
    return 0.5 * Math.log(2 * Math.PI * n) + n * Math.log(n) - n;
  };
  
  const logProb = (shape - 1) * Math.log(x) - x / scale - 
                  shape * Math.log(scale) - logGamma(shape);
  return Math.exp(logProb);
};
```
