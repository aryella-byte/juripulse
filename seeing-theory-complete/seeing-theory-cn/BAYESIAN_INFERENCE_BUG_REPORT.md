## 2. 根本原因分析

### 2.1 错误代码

```typescript
// 第 382-383 行 - ❌ 错误逻辑
const pEvidenceIfHigh = observed ?
  (ev.isPositive ? ev.pIfHigh : 1 - ev.pIfHigh) :
  (ev.isPositive ? 1 - ev.pIfHigh : ev.pIfHigh)

const pEvidenceIfLow = observed ?
  (ev.isPositive ? ev.pIfLow : 1 - ev.pIfLow) :
  (ev.isPositive ? 1 - ev.pIfLow : ev.pIfLow)
```

### 2.2 逻辑错误分解

以"有稳定工作"为例：
```typescript
{ name: '有稳定工作', pIfLow: 0.7, pIfHigh: 0.25, isPositive: false }
```

**参数含义**：
- `pIfLow: 0.7` = 低风险累犯中，70% 有稳定工作
- `pIfHigh: 0.25` = 高风险累犯中，25% 有稳定工作
- `isPositive: false` = 这是降低风险的负面证据

**错误计算过程**：
当 `observed = true` (被告有稳定工作) 时：
```typescript
pEvidenceIfHigh = 1 - ev.pIfHigh = 1 - 0.25 = 0.75  // ❌ 错！
pEvidenceIfLow = 1 - ev.pIfLow = 1 - 0.7 = 0.3      // ❌ 错！
```

**为什么错误**：
- 高风险者有稳定工作的概率应该是 0.25，不是 0.75
- 低风险者有稳定工作的概率应该是 0.7，不是 0.3
- 代码错误地对负面证据取了补概率，导致逻辑完全反转

## 3. 修复方案

### 3.1 修复代码

```typescript
// 第 376-401 行 - ✅ 正确逻辑
// 逐步贝叶斯更新
const riskHistory = [priorRisk]
let currentRisk = priorRisk
for (let i = 0; i < evidenceSteps.length; i++) {
  const ev = evidenceList[i]
  const observed = evidenceSteps[i] // true = 该因素存在

  // P(证据|高风险) - 在高风险累犯中该证据出现的概率
  // P(证据|低风险) - 在低风险累犯中该证据出现的概率
  let pEvidenceIfHigh: number
  let pEvidenceIfLow: number

  if (observed) {
    // 证据存在（选择"是"）
    pEvidenceIfHigh = ev.pIfHigh  // 直接使用高风险下该证据出现的概率
    pEvidenceIfLow = ev.pIfLow    // 直接使用低风险下该证据出现的概率
  } else {
    // 证据不存在（选择"否"）
    pEvidenceIfHigh = 1 - ev.pIfHigh  // 高风险下该证据不出现的概率
    pEvidenceIfLow = 1 - ev.pIfLow    // 低风险下该证据不出现的概率
  }

  // 贝叶斯更新: P(高风险|证据) = P(证据|高风险) * P(高风险) / P(证据)
  const pEvidence = pEvidenceIfHigh * currentRisk + pEvidenceIfLow * (1 - currentRisk)
  currentRisk = (pEvidenceIfHigh * currentRisk) / pEvidence
  riskHistory.push(currentRisk)
}
```

### 3.2 关键改进

1. **简化逻辑**：移除了复杂的三元嵌套运算符
2. **语义清晰**：添加详细注释说明概率含义
3. **直观正确**：
   - 证据存在 → 直接使用原始概率
   - 证据不存在 → 使用补概率 (1 - p)
4. **移除 `isPositive` 依赖**：原代码试图用 `isPositive` 字段控制逻辑，但这是错误的。贝叶斯更新不需要预先区分"正面"或"负面"证据——这是自动从概率计算中得出的。

## 4. 验证结果

### 4.1 数学验证

#### 案例 1：有固定住所
**数据**：`{ pIfLow: 0.8, pIfHigh: 0.4 }`
**初始风险**：30%
**观测**：是

**计算**：
```
P(固定住所|高风险) = 0.4
P(固定住所|低风险) = 0.8
P(固定住所) = 0.4 × 0.3 + 0.8 × 0.7 = 0.12 + 0.56 = 0.68

P(高风险|固定住所) = (0.4 × 0.3) / 0.68 = 0.176 = 17.6%
```

**结果**：✅ 风险从 30% 降至 17.6%（符合预期）

---

#### 案例 2：有稳定工作
**前序风险**：17.6%
**数据**：`{ pIfLow: 0.7, pIfHigh: 0.25 }`
**观测**：是

**计算**：
```
P(稳定工作|高风险) = 0.25
P(稳定工作|低风险) = 0.7
P(稳定工作) = 0.25 × 0.176 + 0.7 × 0.824 = 0.044 + 0.577 = 0.621

P(高风险|稳定工作) = (0.25 × 0.176) / 0.621 = 0.071 = 7.1%
```

**结果**：✅ 风险从 17.6% 降至 7.1%（符合预期）

---

#### 案例 3：曾有暴力前科
**前序风险**：7.1%
**数据**：`{ pIfLow: 0.05, pIfHigh: 0.4, isPositive: true }`
**观测**：是

**计算**：
```
P(暴力前科|高风险) = 0.4
P(暴力前科|低风险) = 0.05
P(暴力前科) = 0.4 × 0.071 + 0.05 × 0.929 = 0.028 + 0.046 = 0.074

P(高风险|暴力前科) = (0.4 × 0.071) / 0.074 = 0.384 = 38.4%
```

**结果**：✅ 风险从 7.1% 升至 38.4%（符合预期，暴力前科确实应该增加风险）

## 8. 附录：贝叶斯更新原理回顾

### 8.1 基本公式

```
P(H|E) = P(E|H) × P(H) / P(E)
```

其中：
- P(H|E)：后验概率（给定证据后的假设概率）
- P(E|H)：似然（假设为真时观察到证据的概率）
- P(H)：先验概率（观察证据前的假设概率）
- P(E)：边缘似然（证据出现的总概率）

### 8.2 在累犯风险评估中的应用

- H = "被告是高风险累犯"
- E = 具体证据（如"有稳定工作"）
- P(E|H=高) = 高风险累犯中该证据出现的概率
- P(E|H=低) = 低风险累犯中该证据出现的概率

**关键洞察**：
- 如果 P(E|高) < P(E|低)，观察到 E 会降低风险
- 如果 P(E|高) > P(E|低)，观察到 E 会增加风险
- 这个"方向"是自然从似然比计算得出的，不需要额外标记！
